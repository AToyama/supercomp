introdução:

Este projeto tem como objetivo realizar um estudo de diferentes técnicas de supercomputação e sua diferença de performance. Para realizar os testes adotamos um modelo de simulação baseado em conceitos explorados em Física do Movimento, em que simulamos a colisão de corpos em um plano 2D.

Modelo de Simulação:

Para nossa simulação consideramos que cada corpo simulado possui massa, aceleração, raio,  velocidade e posição, assim adotamos um modelo elástico para tratar colisão entre dois corpos. Esse modelo obedece a regra de que a quantidade de movimento e a energia cinética se mantém constantes após uma colisão, então se considerarmos uma colisão entre dois objetos de m1, m2 e velocidade v1 e v2, vemos que o momento permanece constante após a colisão na seguinte equação:

m1v1 + m2v2 = m1v1' + m2v2'

Para obtermos a velocidade do objeto após a colisão nos baseamos na seguinte equação

**wikipedia**

nos aproveitamos do fato de que já temos a velocidade dividida em suas componentes direcionais vx e vy e modificamos a equação para cada respectiva direção:

vx
vy
vx2
vy2

Por fim temos a nossa "mesa", onde ocorre a simulação dos corpos, ela tem uma largura, altura e um coeficiente de atrito dinâmico, que é usado para o cálculo da aceleração dos corpos.

||a|| = coeficiente * massa * g

Executando o projeto:

O projeto é compilado através de CMake, ao compilar com o comando "make" são gerados quatro diferentes executaveis no diretório:

	-visualizador -> para ver a simulação gráfica
	-mov_seq -> realizar o teste de tempo da versão sequencial
	-mov_par -> realizar o teste de tempo da versão paralelizada
	-mov_simd -> realziar o teste de tempo com instruções SIMD

Os arquivos devem ser eceutados da seguinte maneira:

	./[executavel] [arquivo de entrada]

Para o arquivo de entrada temos alguns exemplos prontos para testar diferentes situações:

- input1.txt -> teste de um corpo colidindo com a parede
- input2.txt -> teste de colisão entre dois corpos de massa diferente
- input3.txt -> teste de colisão simultânea de três corpos
- input4.txt -> teste de colisão entre vários corpos de diferentes tamanhos e massas

para melhor visualização, nesse específicos testes consideramos o coeficiente de atrito como 0, para podermos observar o comportamento dos corpos por um período maior.
Por fim, temos no projeto o arquivo "input_generator.py", que é basicamente um scrip de python que gera um arquivo chamado "input.txt" no qual se pode pré definir os dados da simulação. Para mudar os dados do arquivo gerado basta mudar os valores no começo do código:

-foto

table_width -> largura da mesa
table_heigth -> altura da mesa
number_of_bodies -> número de corpos a ser simulados
radius -> raio dos corpos (nesse caso todos os corpos sempre terão o mesmo raio)
mu -> coeficiente de atrito dinâmico
speed_range -> atribui uma velocidade aleatória para os corpos entre x e -x
mass -> massa dos corpos
mass_range -> atribui uma massa aleátoria entre 1 e x, se esse atributo for deixado como 0, todos os corpos terão a mesma massa com o valor de "mass".
